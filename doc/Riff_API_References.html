each()

each( function )

Argument

function
선택된 DOM Element를 순회하며 실행될 함수. 이 함수는 실행시 첫번째 인자로 DOM Element의 순서(index)가 전달된다.

Return

riff Object

Description

선택된 DOM Element를 순회하며 실행될 function을 실행한다 첫번째 인자 index로 DOM Element의 순서가 전달되며 함수 내에서의 this는 해당 DOM Element가 된다.

Example

다음은 리스트형식의 마크업이다.

<ul>
	<li>foo</li>
	<li>bar</li>
</ul>

리스트 아이템을 순회하여 각각의 인덱스와 텍스트를 출력한다.

$('li').each(function(index) {
    alert(index + ': ' + $(this).text());
  });

filter()

filter( [selector] )

Argument

selector( optional)
선택된 DOM Element 중 대응되는 DOM Element만 선택하기 위한 String, tagName, Class, ID 의 사용이 가능. (optional)

Return

riff Object

Description

선택된 DOM Element에 대해 selector에 해당하는 DOM Element를 선택하여 반환한다. 인자값이 입력되지 않았거나 String이 아니라면 기존 선택된 riff Object를 반환한다.

Example

$('li').filter(".foo").hide();

"LI" Tag Element를 선택하고 그중 클래스명이 "foo"인 Element에 hide() 를 실행한다.

Issue

id, class, tagName 으로 선택하게 되어 있는 걸 CSS Selector 규칙 모두로 확장 적용할 필요가 있다.

buffer()

buffer( key )

Argument

key
선택된 DOM Element에서 얻을 value 값에 대응하는 String( = 데이터 value 에 대해 key 값으로 지정한 문자열 ).

Return

저장한 값( 숫자, 문자, 객체, 함수 등 )

Description

전역배열인 riffGlobal.buffer 배열에 값을 저장하거나, key 값에 해당하는 값을 불러와 반환한다. 전역 배열의 index값으로는, class마다 고유하게 할당된 riffGlobal.prefixBufferID 값을 이용한다. ( prefix + 숫자( 증가값 ) ) 즉, buffer() 함수가 실행되는 각각의 DOM Element마다 저장공간을 확보하고, 거기에 key - value로 value 값을 저장하거나, 불러온다.
선택된 DOM Element 중 첫번째 DOM Element의 key 값에 대응하는 value를 반환한다. 선택된 DOM Element가 복수개이고 key 값이 설정되면, 선택된 DOM Element 중 맨 첫번째 DOM Element에 대해 key 값으로 value를 반환한다.

buffer( key, value )

Argument

selector (Optional)

option
선택된 DOM Element 중 대응되는 DOM Element만 선택하기 위한 String.tagName, Class, ID 의 사용이 가능. (optional)

Return

Description

전역배열인 riffGlobal.buffer 배열에 값을 저장하거나, key 값에 해당하는 값을 불러와 반환한다. 전역 배열의 index값으로는, class마다 고유하게 할당된 riffGlobal.prefixBufferID 값을 이용한다. ( prefix + 숫자( 증가값 ) ) 즉, buffer() 함수가 실행되는 각각의 DOM Element마다 저장공간을 확보하고, 거기에 key - value로 value 값을 저장하거나, 불러온다.

swipe()

swipe( isFlickEnd , verlocityPercent, fnEventHandle )

Argument

Return

riff Object

Description

swipe event 는 darg, flickRight, flickLeft가 합쳐진 event 이다.
DOM Elenent에 대해 drag event -> flick event 가 발생한다.
이 때 이벤트가 적용중인 DOM Element는, 자연스럽게 움직이며( CSS의 left값 변환) 이동하다 멈춘다.
그러므로 움직이는 DOM Element를 감싸주는 box 역할을 하는 DOM Element가 필요하다.
DOM Element에 swipe 이벤트를 적용한 후 swipe 이벤트가 발생하면, 인자값인 _fnEventHandle 가 작동한다.
_fnEventHandle 에는, swipe의 상태( drag/ 감속이동 / 정지 )를 알 수 있는 인자값을 전달한다.
_fnEventHandle 함수에 전달되는 인자값은 다음과 같다.
_fnEventHandler( state, ptx, pty )
state : swipe event가 적용되어 움직이는 DOM Element의 현재 상황을 나타나는 값.
문자열이며, 다음의 중 한 가지를 반환한다.
"dragMove" : swipe event 중 drag 가 발생중일 때 전달하는 값.
"leftMove" : swipe event가 발생 해서 왼쪽으로 움직이기 시작할 때 최초로 한번 전달하는 값.
"rightMove" : swipe event가 발생 해서 오른쪽으로 움직일 때 최초로 한번 전달하는 값.
"move" : swipe event가 발생하고 움직일 때 전달하는 값.
"leftEnd" : swipe event가 발생하여 왼쪽 끝으로 이동 하였을 때에 전달하는 값.
"rightEnd" : swipe event가 발생하여 오른쪽 끝으로 이동 하였을 때에 전달하는 값.
"end" : swipe event가 끝났을 때에 전달하는 값.
고로, 예를 들어 swipe이 왼쪽에서 오른쪽으로 발생할 경우, state는 다음과 같은 순서로 값을 전달한다.
"dragMove" -> "dragMove" -> ... "dragMove" -> "rightMove" -> "move" -> "move" ... "move" -> "rightEnd" -> "end"
ptx : swipe event 중 객체의 left속성 위치를 나타낸다.
pty : swipe event 중 객체의 top속성 위치를 나타낸다.
